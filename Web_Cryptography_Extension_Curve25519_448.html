Proposal for inclusion of curves 25519 and 448 ECDH and EDDSA algorithms to the Web Cryptography API

First draft.

Introduction

Both curves "curve25519" and "curve448" are designed to allow for fast and secure cryptographic operations. For the curve25519, there exists a written proposal, albeit incomplete https://github.com/trevp/curve25519_webcrypto. There was a formal voting with most votes against this; the main reason was that these curves were not included in the CFRG or TLS standards (an example vote is here https://lists.w3.org/Archives/Public/public-webcrypto/2014Aug/0107.html. Now both curves are included in CFRG standard [ref-rfc7748] and in the TLS 1.3 standard [ref-rfc8446], therefore, it seems that including these curves now in the web crypto API is a reasonable choice. Both of them are Montgomery curves and have birationally equivalent Edwards versions, that leads to adoption EdDSA (Edwards-curve Digital Signature Algorithm [ref-ed25519]) as signature algorithm.
